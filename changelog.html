2026-04-29 UI — Kanban Board Manual — Dave
Kanban outreach board added with five stages, click-to-expand cards, and history notes

New kanban.html page wired into the site nav. Five columns: Follow Up (auto-populated from any card already tagged for follow-up on the Pipeline page), Contacted, Talked To, Accepted, Rejected. Each card shows the company name as a clickable link to the startup URL, a one-line summary, and the BG score; clicking anywhere else on the card opens a full detail modal with the FinServ signal, team, thesis, and evidence list — the same content as the Pipeline-page card.

Each card has a history / notes textarea in the modal — backed by the same note field as the Pipeline page, so notes stay in sync between the two views. A "+ Stamp date" button appends a fresh dated line so the field acts as a running log of outreach activity. Cards can be dragged between columns or moved via stage buttons in the modal; "Remove" clears the card off the board entirely. Stage transitions persist to the existing Turso bg_votes table via a one-shot ALTER TABLE that adds a kanban_stage column on first load (idempotent — swallows the duplicate-column error on subsequent loads).

The 90-startup data array was extracted out of pipeline.html into a shared startups-data.js file so both pages share a single source of truth.

Evidence
New: kanban.html, startups-data.js. Updated: pipeline.html (extracted data), index.html / archive.html / changelog.html (Kanban added to nav). Schema: bg_votes table now carries kanban_stage TEXT (nullable). Existing voting flows untouched — Pipeline INSERT statement does not reference the new column, so kanban_stage is preserved across pipeline saves.

2026-04-15 System Human-initiated
Phase 0 coverage audit added to bg-full-cycle pipeline

New Phase 0 runs at the start of every cycle before any other work. It compares the thesis gap map against the actual pipeline, counting companies per category. Any category with fewer than 3 companies is flagged as "structurally underrepresented" and automatically elevated to critical scan priority. Any category with 0 companies is flagged as a "blind spot" and receives mandatory search allocation.

This prevents the system from endlessly deepening coverage in categories it already knows about while ignoring categories it has never searched — the structural failure that caused the platform infrastructure exposure bias.

Evidence
The v3.0 rebalance revealed that 9 of 11 specialist agent types from the thesis had zero pipeline companies, zero search terms, and zero scoring signals. Exposure normalisation and thesis anchoring catch scoring and preference drift, but neither catches the upstream problem: categories that were never searched in the first place. Phase 0 closes this gap.
