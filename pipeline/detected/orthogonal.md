# Orthogonal

**URL:** https://www.ycombinator.com/companies/orthogonal
**Gap:** P7 — Agent Monetisation
**Stage:** Pre-Seed (YC W26)
**Geo:** US
**Score:** 52

## One-liner
Pay-per-use API marketplace enabling AI agents to autonomously discover, access, and pay for APIs without subscriptions or human intervention.

## FinServ Signal
Weak direct finserv signal. Founders have ex-Coinbase (payments) and Google APIs backgrounds. The product is a general API marketplace, not finserv-specific. However, the underlying infrastructure — agents paying APIs autonomously on a per-use basis — is directly applicable to the financial agent marketplace thesis where specialist agents call research, data, compliance, and execution APIs.

## Team
- **Christian Pickett (co-founder):** Payments at Coinbase + Google APIs experience.
- **Bera Sogut (co-founder):** Technical co-founder.
- **Investors:** Y Combinator (W26). 20+ API partners already integrated.

## Thesis
Orthogonal is building the monetisation layer the app-store-for-agents needs: a marketplace where any API provider can list once and get paid automatically every time an agent calls. This is the plumbing for the P7 (Agent Monetisation) gap. The main scoring concern is that it's a horizontal play, not finserv-specific. A finserv-native version of Orthogonal (or Orthogonal expanding into financial data APIs specifically) would score much higher. Track as a P7 reference architecture; the gap still needs a finserv-focused entrant. Scored down for non-FinServ focus (–10 per preferences).
