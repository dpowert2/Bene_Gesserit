## One-liner
Identity infrastructure that gives AI agents a verified credential (Agent ID) anchored to a confirmed human owner (Alien ID via facial recognition + social graph), solving the accountability gap in autonomous agent commerce.

## FinServ Signal
The core problem Alien solves is directly relevant to financial services: when an AI agent initiates a transaction, who is legally responsible, and how do you prove it was authorised by a verified human? This maps precisely to the KYA (Know Your Agent) problem that every regulated financial institution will face as agentic AI deploys into client workflows. Alien does not retain biometric data permanently — it confirms human identity then layers probabilistic scoring via social graph, producing a portable Agent ID credential. Investors include Initialized Capital, Finality, and Mantaray. The product architecture (human-anchored agent credential) could become the standard IAM layer for agent-to-agent financial transactions, especially where AML/KYC obligations require a human counterparty to be identifiable.

## Thesis
Financial agents acting autonomously — placing orders, initiating payments, executing diligence queries — create a new principal problem: the traditional KYC/AML chain assumes a human initiates every regulated action. Alien proposes to solve this by making every agent traceable to a verified human from day one, via a portable credential (Agent ID) rather than session-by-session verification. In financial services, this matters because regulators will eventually require demonstrable human authorisation chains for automated transactions. Alien builds that chain at the identity layer rather than the application layer. Gap fit is real but broad — Alien is a horizontal identity infrastructure play, not finserv-native. Score reflects this; the core architecture (agent identity anchored to human) is the right solution to an emerging finserv problem even if current market positioning is generic.
